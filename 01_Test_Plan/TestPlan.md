# Test Plan — TaskFlow (Web)

## 1. Overview
**Product:** TaskFlow — web-приложение для управления задачами (доски/колонки/карточки).  
**Goal:** обеспечить корректность ключевых пользовательских сценариев, стабильность UI и API, предотвращение потери данных.

## 2. Scope

### In scope (тестируем)
- Регистрация/логин/логаут
- Профиль пользователя (минимум: просмотр, выход)
- Доски (создание/переименование/удаление)
- Колонки (создание/переименование/удаление)
- Карточки задач (создание/редактирование/удаление)
- Перемещение карточек между колонками (Drag&Drop / кнопками — зависит от реализации)
- Поиск/фильтр задач (если есть)
- Базовая валидация форм (пустые поля, длины, формат email)
- API (основные методы CRUD)
- Кроссбраузерность (Chrome, Firefox)

### Out of scope (не тестируем в рамках портфолио)
- Нагрузочное тестирование
- Глубокое security-тестирование (пентест)
- Полная локализация/мультиязычность
- Accessibility-аудит уровня WCAG (только базовые проверки)

## 3. Test Approach

### Types of testing
- Smoke
- Functional
- UI testing
- Regression
- API testing
- Exploratory testing
- Basic cross-browser testing

### Test design techniques
- Equivalence partitioning
- Boundary values
- Error guessing
- State transition (статусы карточек)

## 4. Test Environment
- **Staging:** локально / тестовый стенд (описать, когда появится)
- **Browsers:** Chrome (latest), Firefox (latest)
- **OS:** Windows 10/11

## 5. Entry / Exit Criteria
### Entry
- доступен билд/стенд
- основные требования описаны (минимум в README продукта)
- есть тестовые аккаунты/данные

### Exit
- Smoke проходит
- Нет блокирующих багов (Critical/High) по ключевым сценариям
- Заведены баг-репорты на все найденные дефекты

## 6. Risks
- Потеря задач при обновлении страницы
- Дублирование карточек при двойном клике/повторной отправке формы
- Некорректная авторизация/сессии
- Несогласованность UI и API (UI показывает одно, API сохраняет другое)

## 7. Deliverables
- Test Plan
- Checklists (Smoke + Regression)
- Test Cases (таблица)
- Bug Reports (шаблон + примеры)
- Postman collection + краткое описание API проверок
- SQL проверки (если есть БД-дамп/схема)

## 8. Roles & Responsibilities (для портфолио)
- QA Engineer: анализ требований, подготовка тест-документации, проведение тестирования, заведение багов, отчётность.
